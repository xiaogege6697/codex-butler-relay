# Changelog

## 0.3.0 - 2026-07-03

### 新功能

- 新增 `--detach`：后台等待 Claude 终态，派发后立即结束 Codex 当前回合。
- 新增 `--collect`：读取终态；任务仍运行时把下一次检查间隔从 10 分钟开始依次翻倍。
- 终态到达时发送 macOS 通知；后台结果与日志保存在用户 cache，不污染项目源码。

### 优化

- Codex Skill 默认使用同线程 heartbeat：首次 10 分钟检查，未完成按 20、40、80 分钟指数退避；运行中保持静默。
- 项目定位只使用当前工作目录或用户明确路径，不再扫描 memory 与整个用户目录。

## 0.2.6 - 2026-07-03

### 优化

- Goal 包装压缩为目标、授权边界和终态接口，不再预先展开任务拆分、模型路由或派发证明。
- Butler 自主判断是否调用低成本临时工；不为流程合规强制外包或生成清单。
- 兼容最终一行以终态信号开头并紧跟摘要，避免额外模型纠正轮次。

## 0.2.5 - 2026-07-03

### 修复

- 容忍终态行前最多 4 个短前言行，避免为已完成结果额外调用一次强模型做协议纠正。

## 0.2.4 - 2026-07-03

### 修复

- 长 Goal 按 UTF-8 字节安全分块注入 GNU screen，避免 `stuff` 静默丢弃整段中文任务。
- 在前 5 个非空行内识别独立终态行，容忍模型的极短前言而不误判正文。

## 0.2.3 - 2026-07-03

### 修复

- 增加首条消息投递握手：只有 transcript 出现同一 relay marker 才进入长任务等待。
- 自动补发被吞掉的 Enter，必要时重发消息；三次失败后明确报错，不再无限盲等。

## 0.2.2 - 2026-07-03

### 修复

- 交互式 Claude 默认启用 `auto` permission mode，常规项目内工具审批不再打断 Goal Loop。
- Codex 在 Relay 健康运行时禁止轮询与普通进度复述，只在终态、错误或用户明确查询时介入。

## 0.2.1 - 2026-07-03

### 修复

- 自动识别 Claude Code 首次进入项目时的目录信任提示，并仅在默认信任选项存在时确认一次。
- 在注入长文本后短暂等待再提交，避免 Claude TUI 偶发吞掉 Enter。

## 0.2.0 - 2026-07-03

- 新 Goal 自动使用新 session，仅首次加载 `/butler`；同 session 续接只转发文本。
- 普通进度留在前台 TUI，仅以 `GOAL_DONE`、`NEED_DECISION`、`NEW_WINDOW` 触发关键节点接力。
- 明确临时工的经济性门槛：执行、审核与预期返工总成本更低，且结果可验证。
- 换窗后安全关闭被替换的 `butler-native-*` screen。
- 删除 `--native`、`--visible`、`--foreground`、`--adopt`；默认即真实交互式前台，后台仅保留显式 `--headless`。

## 0.1.0 - 2026-07-02

- 首个公开版本：文本转发、会话续接、状态监测、自动换窗与 Goal 验收闭环。
